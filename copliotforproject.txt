What python -m <module-or-package> does

Runs a library module or package as a script. Python locates the module on sys.path, loads it, and executes it as if it were the main module.
Useful because it runs the module with the selected Python interpreter (useful for virtual environments), and it finds modules by package name instead of needing the file path.
Common uses:
Run simple tools built into Python: python -m http.server
Run pip with the current interpreter: python -m pip install requests
Run your own module or package that exposes a CLI via if __name__ == "__main__": or a __main__.py inside a package.
How Python decides what to run

If you give a module name (e.g., mymodule), Python imports it and executes its top-level code. If that module has:
if name == "main":
...
that block runs.
If you give a package name (e.g., mypackage), Python looks for mypackage.__main__ (a module or package __main__.py) and executes that. This is how packages expose a CLI.
Behavior details

sys.argv: when you run python -m modulename arg1 arg2, the module sees sys.argv[0] == modulename (not the filename); args follow in sys.argv[1:].
package: modules run with -m get correct package context so relative imports inside packages work.
Use python -m pip instead of pip to be sure you're calling the pip matching that Python interpreter.

```powershell
# find PID(s) using port 8080

taskkill /PID 1234 /F
```

```powershell
Stop-Process -Name python -Force
```

```powershell
Get-Job
```

Examples (PowerShell)
- Run a quick HTTP server on port 8000:
```powershell
python -m http.server 8000
```
Stop it with Ctrl+C in the same terminal.

- Install a package using the current Python:
```powershell
python -m pip install requests
```

- Run a module with arguments:
```powershell
python -m some_module arg1 arg2
```

- Serve a directory and specify a bind address:
```powershell
python -m http.server 8000 --bind 127.0.0.1
```

- `mypkg/__init__.py` can be empty or have helpers.
- `mypkg/__main__.py`:
```python
import sys
print("mypkg __main__ running, argv:", sys.argv)
```
Run it:
```powershell
python -m mypkg a b c
```
That executes `mypkg.__main__`.

Why use -m frequently
- Ensures correct interpreter/venv (especially for pip).
- Lets you run installed packages by name.
- Enables proper package-relative imports and correct sys.path setup when running packages.

Advanced: run a module from a package path
- You can run a submodule using the `package.module` notation:
```powershell
python -m package.module args...
```
This imports and runs `package.module` as __main__ (so the module's `if __name__ == "__main__":` runs). Be careful with relative imports inside that module.

- Get the PID that owns port 8080 and show what process it is:
```powershell
# find PID listening on 8080
$pid = (Get-NetTCPConnection -LocalPort 8080 -State Listen).OwningProcess
# show the process (if any)
if ($pid) { Get-Process -Id $pid | Format-Table Id, ProcessName, Path -Auto } else { "No listening process found on port 8080" }
```
- If that shows the process you want to stop, kill it:
```powershell
Stop-Process -Id $pid -Force
```

Alternative (works on older Windows where Get-NetTCPConnection might not show the listener)
- Use netstat to get the PID and then taskkill:
```powershell
# find PID(s) using port 8080
netstat -ano | Select-String ":8080"

# suppose the PID printed is 1234, then:
taskkill /PID 1234 /F
```

If you know it's a Python server and want to kill all Python instances (careful: this will stop all Python processes)
```powershell
Stop-Process -Name python -Force
```
